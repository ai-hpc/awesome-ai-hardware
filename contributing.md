[Contributor Code of Conduct](code-of-conduct.md). By participating in this

## Adding something to this list

You will need a GitHub account! If you are new to GitHub, [get started here](https://github.com/join).

1. Access the [awesome-ai-hardware GitHub page](https://github.com/ai-hpc/awesome-ai-hardware).
2. Click on the `readme.md` file.
3. Click on the edit icon.
4. Edit the file in the in-browser editor using [GitHub Flavored Markdown](https://help.github.com/articles/github-flavored-markdown/).
5. Describe why you're proposing the change, then click "Propose file change".
6. Submit the pull request!

## Guidelines

Ensure your pull request adheres to the following:

### General

- Search previous suggestions before making a new one to avoid duplicates.
- Make sure the resource is genuinely awesome — not just OK or average.
- Make an individual pull request for each suggestion.
- Use the format: `[Resource Name](link) - Description.`
- Additions should be placed at the bottom of the relevant category.
- Keep descriptions short, objective, and non-promotional.
- Descriptions must start with a capital letter and end with a period.
- New categories are welcome if they contain at least 3 items.
- Check your spelling and grammar.
- The pull request title should be: `Add [Resource Name]`.

### What belongs here

This list covers resources related to **AI hardware** — chips, accelerators, compilers, profiling tools, and research about running ML workloads on specialized hardware. Specifically:

- **Hardware Platforms** — GPUs, AI accelerators (TPUs, IPUs, LPUs, etc.), edge/embedded AI chips, and their SDKs or programming models.
- **Compilers & Runtimes** — Tools that compile, optimize, or execute ML models on hardware (XLA, TVM, Triton, TensorRT, etc.).
- **Benchmarks & Profiling** — Tools and suites for measuring hardware performance or profiling ML workloads.
- **Tutorials & Courses** — Hands-on learning resources focused on programming AI hardware or understanding its internals.
- **Papers** — Influential research on hardware architecture, memory systems, compilers, or inference efficiency. Prefer peer-reviewed or widely cited works.
- **Books** — In-depth books covering GPU programming, parallel computing, or efficient ML.
- **Community** — Active forums, Discord servers, or publications focused on AI hardware.

### What does NOT belong here

- General ML/AI frameworks or libraries unless they are specifically relevant to hardware programming or optimization.
- Vendor marketing pages with no technical depth.
- Unmaintained, archived, or deprecated projects.
- Blog posts or one-off articles (prefer official documentation or tutorial series).
- Papers without meaningful hardware focus.
